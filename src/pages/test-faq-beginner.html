🚫 Permission denied 오류가 나요

터미널이란?

마우스 대신 키보드로 컴퓨터에게 명령을 내리는 프로그램입니다.

왜 써야 하나요? Claude Code는 개발 도구라서 터미널을 통해 작동합니다. 하지만 걱정 마세요!

  • 복사-붙여넣기만 하면 됨: 명령어를 외울 필요 없어요
  • Claude가 친절히 설명: 무슨 일이 일어나는지 알려줌
  • 실수해도 괜찮아: 컴퓨터가 망가지지 않아요

터미널 여는 법

  • Command (⌘) + Space → "터미널" 또는 "Terminal" 검색 → Enter

명령 프롬프트/Git Bash란?

키보드로 컴퓨터에게 명령을 내리는 프로그램들입니다.

어떤 차이가 있나요?

  • 명령 프롬프트: Windows 기본 프로그램 (프로그램 설치용)
  • Git Bash: Mac 터미널과 비슷하게 작동 (실제 개발용, 더 편리함)

🎯 Windows 사용자 팁

Git for Windows 설치 후에는 Git Bash를 주로 사용하게 됩니다. Mac의 터미널과 비슷하게 작동해서 편리해요!

Claude 계정은 어디서 만드나요?

Claude 계정 만들기: claude.ai에서 회원가입하세요.

📝 회원가입 단계

  • 이메일 주소 입력 (구글 계정으로도 가능)
  • 인증번호 확인 (이메일 또는 SMS)
  • 이름과 용도 입력 (간단한 설문)
  • 완료! 바로 사용 가능

💰 요금제 선택

  • Free 플랜: 무료, 하루 몇 번 사용 가능 (처음 배울 때 충분)
  • Pro 플랜: $20/월, 5배 더 많은 대화, Claude 3 Opus 및 Haiku 모델 사용 가능

💡 추천: 처음에는 Free로 시작해서 감을 잡은 후 업그레이드하세요!

내 컴퓨터 사양으로 가능한가요?

🍎 Mac 요구사항

  • macOS 10.15 이상 (거의 모든 Mac 가능)
  • Intel Mac: 완벽 지원 ✅
  • Apple Silicon (M1, M2, M3, M4, Pro, Max): 완벽 지원 ✅
  • 저장공간: 약 500MB (매우 적음)
  • 메모리: 4GB 이상 권장

✨ 2019년 이후 Mac이라면 문제없이 작동합니다!

🪟 Windows 요구사항

  • Windows 10 버전 2004 이상 또는 Windows 11
  • 64bit만 지원 (32bit 안됨)
  • 저장공간: 약 1GB (Git + Node.js + Claude Code)
  • 메모리: 4GB 이상 권장

🎉 좋은 소식: Claude Code는 Windows에서도 네이티브로 실행됩니다!

🔍 내 컴퓨터 사양 확인법

Apple 메뉴 → 이 Mac에 관하여에서 확인

Windows 키 + i → 시스템에서 확인

환경 설치 중 환경 설치 중

Homebrew, Node.js Git for Windows, Node.js

환경 설치 중 가장 많이 발생하는 문제들과 해결법을 정리했습니다. 걱정 마세요, 모두 해결 가능해요!

🔐 비밀번호를 입력하는데 화면에 안 나와요!

언제 이런 일이 생기나요? Homebrew 설치할 때 비밀번호를 입력하라고 하는데 화면에 아무것도 안 나타나요.

🖥️ 실제 터미널 화면
❯ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" ==> This script will install: /opt/homebrew/bin/brew ... Password: ← 여기서 입력해도 화면에 안 보임!

✅ 해결법: 정상이니까 그냥 입력하세요!

  1. 비밀번호 입력: 화면에 안 보여도 입력되고 있어요
  2. Enter 누르기: 다 입력했으면 Enter
  3. 틀렸다면: "Sorry, try again." 나오면 다시 입력

💡 왜 이럴까요? 보안상 비밀번호가 화면에 표시되지 않는 것이 정상입니다!

언제 이런 일이 생기나요? Git for Windows나 Node.js 설치할 때 "관리자 권한으로 실행하시겠습니까?" 창이 뜨는데 비밀번호를 입력하라고 해요.

🔒 Windows 관리자 권한 요청

프로그램을 설치할 때 Windows가 "정말 설치하시겠습니까?"라고 물어보는 것입니다.

  1. 팝업창 확인: "사용자 계정 컨트롤" 창이 뜸
  2. 비밀번호 입력: Windows 로그인 비밀번호 입력
  3. "예" 클릭: 설치 진행

💡 비밀번호가 없다면?

Windows 로그인 시 비밀번호를 설정하지 않았다면 그냥 "예"를 클릭하면 됩니다.

😰 영어로 뭔가 물어보는데 뭘 선택해야 하나요?

자주 나오는 영어 메시지들과 답변법:

📝 자주 나오는 질문들
Press RETURN/ENTER to continue or any other key to abort: → 엔터를 누르면 계속 진행!

Do you want to continue? [Y/n] → Y를 입력하고 엔터!

Would you like to install command line developer tools now? (y/N) → y를 입력하고 엔터!

✅ 황금 규칙

  • "Continue?" → Y 입력
  • "Install?" → Y 입력
  • "Agree?" → Y 입력
  • "Press ENTER" → 엔터 누르기

🎯 핵심: 거의 모든 질문에 "예(Y)"라고 답하면 됩니다!

⚠️ 실수로 N을 눌렀다면?

설치가 중단됩니다. 다시 명령어를 실행해서 처음부터 시작하세요. 컴퓨터가 망가지지 않으니 걱정 마세요!

⏰ 설치가 5분째 진행 중인데 멈춘 건가요?

이런 화면을 보고 계신가요?

🖥️ 설치 진행 중 화면
⠸ reify:typescript: timing reifyNode Completed in 5234ms ← 5분째 이 화면... 멈춘 거 아님! ⠹ reify:@types/node: http fetch GET 200 ⠼ reify:lodash: timing reifyNode Completed in 3421ms ... (계속 진행 중)

😌 정상입니다! 기다려주세요

  • 소요시간: 5-10분 정도 걸릴 수 있어요
  • 인터넷 속도: 느린 인터넷이면 더 오래 걸림
  • 진행 확인: 글자가 계속 바뀌고 있으면 진행 중
  • 스피너: ⠸⠹⠼⠽ 이런 기호가 돌아가면 작동 중

🚨 정말 멈춘 경우

이런 상황이면 멈춘 거예요:

  • 10분 이상 아무것도 안 바뀜
  • 스피너도 안 돌아감
  • 에러 메시지가 나옴

해결법: Ctrl + C로 중단하고 다시 시도하세요.

💡 기다리는 동안 할 일

커피 한 잔 마시거나 다른 일을 하세요! 컴퓨터가 열심히 일하고 있어요. 🍕

Claude Code 설치 중

npm install 명령어 실행

npm install 명령어를 실행할 때 생기는 문제들입니다. 대부분 권한이나 속도 문제예요.

🚫 Permission denied 오류가 나요

이런 오류 메시지를 보셨나요?

💥 오류 메시지
❯ npm install -g @anthropic-ai/claude-code npm ERR! code EACCES npm ERR! syscall mkdir npm ERR! path /usr/local/lib/node_modules npm ERR! errno -13 npm ERR! Error: EACCES: permission denied ← 권한이 없다는 오류!

✅ Mac 해결법: sudo 사용

1단계: 명령어 앞에 sudo를 붙이세요:

sudo npm install -g @anthropic-ai/claude-code

2단계: 비밀번호 입력 (화면에 안 보임)

3단계: Enter 누르기

💡 sudo란? "관리자 권한으로 실행하라"는 뜻입니다.

✅ Windows 해결법: 관리자 권한으로 실행

  1. Git Bash 닫기: 현재 창을 닫으세요
  2. Git Bash 우클릭: 시작 메뉴에서 Git Bash를 찾아서 우클릭
  3. "관리자 권한으로 실행" 선택: 클릭
  4. 명령어 다시 실행: npm install -g @anthropic-ai/claude-code

🤔 왜 이런 오류가 생기나요?

프로그램을 전역(모든 사용자가 사용할 수 있게)으로 설치하려면 관리자 권한이 필요하기 때문입니다. 보안상 정상적인 절차예요!

🐌 npm install이 너무 오래 걸려요

npm install은 원래 시간이 오래 걸려요!

⏱️ 예상 소요시간

  • 빠른 인터넷: 2-3분
  • 보통 인터넷: 5-7분
  • 느린 인터넷: 10-15분
  • 회사/학교 인터넷: 방화벽 때문에 더 오래 걸릴 수 있음

✅ 정상 진행 확인법

이런 것들이 보이면 정상적으로 설치 중이에요:

  • 📦 패키지 이름들이 계속 바뀜
  • ⠸⠹⠼⠽ 스피너가 돌아감
  • 숫자(ms, KB, MB)가 계속 변경됨
  • "downloading", "extracting", "building" 등의 단어가 나옴

💡 기다리는 동안 할 일

  • ☕ 커피나 차 한 잔
  • 📱 다른 일 하기 (단, 컴퓨터는 건드리지 말기)
  • 🎵 음악 듣기
  • 📖 Claude Code 사용법 미리 읽어보기

🚫 하지 말 것: 터미널 창 닫기, 다른 프로그램 많이 실행하기

📋 명령어가 너무 길어서 복사-붙여넣기가 안 돼요

긴 명령어: npm install -g @anthropic-ai/claude-code

한 글자씩 입력하기에는 너무 길고 실수하기 쉽죠. 복사-붙여넣기가 답입니다!

🍎 Mac 복사-붙여넣기

  1. 명령어 선택: 마우스로 드래그해서 명령어 전체 선택
  2. 복사: ⌘ Command + C
  3. 터미널 클릭: 터미널 창을 클릭해서 활성화
  4. 붙여넣기: ⌘ Command + V
  5. Enter: 명령어 실행

🪟 Windows 복사-붙여넣기

방법 1: 키보드 (추천)

  1. 명령어 선택: 마우스로 드래그해서 명령어 전체 선택
  2. 복사: Ctrl + C
  3. Git Bash 클릭: Git Bash 창을 클릭
  4. 붙여넣기: Ctrl + Shift + V

방법 2: 마우스

  1. 명령어 복사 (Ctrl+C)
  2. Git Bash에서 우클릭
  3. "Paste" 선택

🚨 자주하는 실수

  • 명령어 일부만 복사: 전체를 다 선택했는지 확인
  • 앞뒤 공백 포함: 명령어만 깔끔하게 선택
  • Ctrl+V 사용: Git Bash에서는 Ctrl+Shift+V 사용!
  • Enter 안 누름: 붙여넣기 후 반드시 Enter

💡 프로 팁

명령어를 메모장이나 노트 앱에 미리 저장해두면 나중에 다시 사용할 때 편해요!

계정 연결 중

Claude 계정과 연결

Claude 계정과 연결하는 과정에서 생기는 문제들입니다. 브라우저와 터미널 사이를 오가며 진행해요.

🌐 브라우저가 안 열려요

claude-code auth login 명령어를 실행했는데 브라우저가 자동으로 안 열리나요?

✅ 해결법 1: 수동으로 열기

  1. 터미널 확인: 로그인 URL이 나타났는지 확인
  2. URL 복사: https://로 시작하는 긴 주소 복사
  3. 브라우저 열기: 직접 브라우저를 열어서 주소창에 붙여넣기
  4. 로그인: Claude 계정으로 로그인

✅ 해결법 2: 브라우저 설정

기본 브라우저가 설정되지 않은 경우:

  1. 시스템 환경설정 → 일반
  2. "기본 웹 브라우저" 설정
  3. Chrome, Safari, Firefox 중 선택
  1. 설정 → 앱 → 기본 앱
  2. "웹 브라우저" 클릭
  3. Chrome, Edge, Firefox 중 선택

🔍 터미널에서 찾을 URL 예시

https://claude.ai/login?next=%2Fauth%2Fcli...

이런 긴 주소를 찾아서 복사하세요!

🤔 어떤 계정으로 로그인해야 하나요?

✅ Claude 계정 사용

claude.ai에서 만든 계정을 사용하세요:

  • claude.ai에서 회원가입한 이메일/비밀번호
  • 또는 Google 계정으로 로그인 (claude.ai에서 Google로 가입했다면)
  • 또는 GitHub 계정으로 로그인 (claude.ai에서 GitHub로 가입했다면)

❌ 이런 계정은 안 됩니다

  • OpenAI 계정: ChatGPT용 계정
  • Microsoft 계정: Copilot용 계정
  • Anthropic 직원 계정: 일반 사용자용이 아님

🆕 계정이 없다면?

  1. claude.ai 방문
  2. "Sign up" 또는 "회원가입" 클릭
  3. 이메일 주소 입력하고 인증
  4. 간단한 설문 작성
  5. 계정 생성 완료!

💡 헷갈리지 않는 방법

브라우저에서 claude.ai에 먼저 로그인해보세요. 성공적으로 로그인되면 그 계정이 맞습니다!

✅ 로그인했는데 다음에 뭘 해야 하나요?

브라우저에서 로그인 성공 후:

✅ 정상적인 흐름

  1. 브라우저: "로그인 성공" 또는 "인증 완료" 메시지 표시
  2. 터미널 확인: 터미널로 돌아가서 확인
  3. 성공 메시지: "Successfully logged in" 또는 비슷한 메시지
  4. 브라우저 닫기: 이제 브라우저 탭을 닫아도 됨

🔍 터미널에서 확인할 메시지들

터미널에서 확인할 메시지들
✓ Successfully authenticated with Claude ✓ Logged in as: your-email@example.com You can now use claude-code!

❓ 성공했는지 확인하는 방법

테스트 명령어 실행:

claude-code --version

버전 번호가 나오면 성공! (예: claude-code 1.2.3)

🚨 문제가 있다면?

  • 터미널에 오류 메시지: 다시 claude-code auth login 실행
  • 브라우저가 응답 없음: 페이지 새로고침 후 재시도
  • 권한 오류: 계정 플랜 확인 (Free/Pro)

  2. 로그인 확인: claude-code auth status
  3. 인터넷 확인: 브라우저로 claude.ai 접속 테스트
  4. 폴더 확인: 읽기/쓰기 권한이 있는 폴더인지 확인
